Speeding Up Verification Process

Company Overview

Tanijoy is a startup wich providing an Agriculture Peer-to-Peer Lending Platform -- Most people also know it as an online agriculture investment platform. This platform helps connecting farmers to investors and vice versa. Investors are able to fund farmer projects and get the return. Farmers are also able to get fund from investors to run their farming projects and sell the crop to the market.

Brief

Verifying Investor‚Äôs transactions through the SQL database manually was the ultimate pain for our one and only Customer Service and Admin. She cannot verify transactions unless she involves the IT Team as the only one who can access the database to verify them. It was also a burden for the IT Team since they need to do the extra job.

On the other hand, OJK (Indonesian Financial Services Authority) required some features to be added to our ecosystems, such as investor-farmer data verification, withdrawal, and top-up balance. They would need the other endpoint for our CS Admin to verify investor-farmer data and withdrawal transactions.

So how can we help our CS Admin to verify all transactions and investor-farmer data in an effective way?

Role: UI/UX Designer (Research and Design)

Target User: Customer Service and Admin (CS Admin)

Tools: Sticky notes, Figma, Google Form, Google Docs

Language: Indonesian

Understanding user needs

We interviewed our CS Admin to find out her needs and collect some insights. Then for extracting the interview data into user needs, we used Affinity Diagramming.

Here are user needs we found from the interview:

I need to get Investor‚Äôs data & verify incoming transaction easily without opening SQL Database

Understanding business needs

As we mentioned before, OJK required our company to add some features into our product ecosystem according to their new regulations, including investor-farmer data verification, withdrawal, and top-up balance. Therefore, our CS Admin will need to do some things, such as:

Verify withdrawal transaction
Verify new Investor and new Farmer data

üí° Ideation

Defining Main Features

We discussed some ideas with the product owner to find out the right solutions. Since user and business needs are pretty clear, these are the main features we will carry out :

  1. Transaction verification
  2. Investor data verification
  3. Farmers data verification

Designing Information Architecture

In order to find out the hierarchy of feature and functionality based on the user's perspective, we used the Card Sorting method. We invited CS Admin to a card sorting workshop. Some features and functionalities will be sorted based on the frequency of use and importance. Then we translated the card sorting result into Information Architecture.

Sketches

We rapidly sketched some alternative pages on the window. This lets us explore some various ideas.

üñ• High Fidelity Design

We turned Main Features, Information Architecture, and some Sketches into these delightful high fidelity design. Since we need this product very soon, all these design would be implemented immediately without any user testing.

Dashboard Home:

A shortcut for verifying everything

We show all total of user main tasks at the top of the screen to encourage user verifying investment transactions, withdrawal transactions, new investors, and new farmers

  • We display list groups of all main tasks on the home screen to ease user performing all tasks immediately

Transaction Page:

Elevating the Transaction Verification Effectiveness

  • In order to advance the effectiveness transaction verification, we show lists of all incoming transactions on this page with action buttons provided
  • Transaction lists are shown based on types to keep user focus while dealing with each transaction type.
  • We also provide search and filter to help user finds some particular transactions

Investor Page:

Verify & Get Investor‚Äôs Data

We show the list of all investors who waiting for verification, including the verification action button to support user verifying new investors

  • The total amount of waiting for verification is shown on the top to motivate users verifying new investors
  • We provide search as well to speed up user to find specific investors
  • All investor data is displayed to ease users to check them before verifying the investor.
  • In order to get verified investor‚Äôs data with ease, we also show the list of all verified investors including their personal data.

Farmer Page:

Verify Incoming Farmers

  • The list of incoming new farmers is emphasized on the top of the screen so the user can promptly verify them.
  • We show the list of all farmers based on type including the detailed personal information. This helps the user to get farmer‚Äôs personal information to be contacted.
  • Search is also provided so users are able to find specific farmers easily.

üåª Impact

All transactions can be verified faster and easier after CS Dashboard has been launched. Before we had a CS Dashboard, the user used to involve the IT Team to verify all transactions manually through the database. However, now the transaction verification process has been cut off so the user is able to verify transactions by himself without counting in other roles.

User now can get investor detail information easier. Like verifying transactions, user used to count in the IT Team to get investor information data through the database manually. But now the user is able to view investor data by herself through CS Dashboard.
